Panell d'Exercicis: XML, DTD i XSD

Exercici 1 - Catàleg de llibres

Crea un DTD i un document XML per a representar un catàleg de llibres.
Condicions:
El catàleg pot contenir un o més llibres.
Cada llibre ha de tenir:
titol
autor
any
preu
Cada llibre té un atribut categoria amb valors possibles:
novella
assaig
tecnologia
<!ELEMENT cataleg (llibre+)> <!ELEMENT llibre (titol, autor, any, preu)> <!ATTLIST llibre categoria (novel·la | assaig | tecnologia) #REQUIRED> <!ELEMENT titol (#PCDATA)> <!ELEMENT autor (#PCDATA)> <!ELEMENT any (#PCDATA)> <!ELEMENT preu (#PCDATA)>
<?xml version="1.0" encoding="UTF-8"?> <xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema"> <xs:element name="cataleg"> <xs:complexType> <xs:sequence> <xs:element name="llibre" maxOccurs="unbounded"> <xs:complexType> <xs:sequence> <xs:element name="titol" type="xs:string"/> <xs:element name="autor" type="xs:string"/> <xs:element name="any" type="xs:integer"/> <xs:element name="preu" type="xs:decimal"/> </xs:sequence> <xs:attribute name="categoria" use="required"> <xs:simpleType> <xs:restriction base="xs:string"> <xs:enumeration value="novel·la"/> <xs:enumeration value="assaig"/> <xs:enumeration value="tecnologia"/> </xs:restriction> </xs:simpleType> </xs:attribute> </xs:complexType> </xs:element> </xs:sequence> </xs:complexType> </xs:element> </xs:schema>

Exercici 2 - Agenda de contactes

Crea un XML i un DTD per a representar una agenda de contactes.
Condicions:
L'arrel és agenda.
Pot contenir diversos contactes.
Cada contacte té:
nom
telefon
email (opcional)
L'atribut tipus pot ser:
personal
professional
<!ELEMENT agenda (contacte+)> <!ELEMENT contacte (nom, telefon, email?)> <!ATTLIST contacte tipus (personal | professional) #REQUIRED> <!ELEMENT nom (#PCDATA)> <!ELEMENT telefon (#PCDATA)> <!ELEMENT email (#PCDATA)>
<?xml version="1.0" encoding="UTF-8"?> <xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema"> <xs:element name="agenda"> <xs:complexType> <xs:sequence> <xs:element name="contacte" maxOccurs="unbounded"> <xs:complexType> <xs:sequence> <xs:element name="nom" type="xs:string"/> <xs:element name="telefon" type="xs:string"/> <xs:element name="email" type="xs:string" minOccurs="0"/> </xs:sequence> <xs:attribute name="tipus" use="required"> <xs:simpleType> <xs:restriction base="xs:string"> <xs:enumeration value="personal"/> <xs:enumeration value="professional"/> </xs:restriction> </xs:simpleType> </xs:attribute> </xs:complexType> </xs:element> </xs:sequence> </xs:complexType> </xs:element> </xs:schema>

Exercici 3 - Botiga online

Defineix un XML i un DTD per a representar una botiga online.
Condicions:
L'arrel és botiga.
Conté diversos producte.
Cada producte té:
nom
preu
stock
Atribut id obligatori.
<!ELEMENT botiga (producte+)> <!ELEMENT producte (nom, preu, stock)> <!ATTLIST producte id ID #REQUIRED> <!ELEMENT nom (#PCDATA)> <!ELEMENT preu (#PCDATA)> <!ELEMENT stock (#PCDATA)>
<?xml version="1.0" encoding="UTF-8"?> <xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema"> <xs:element name="botiga"> <xs:complexType> <xs:sequence> <xs:element name="producte" maxOccurs="unbounded"> <xs:complexType> <xs:sequence> <xs:element name="nom" type="xs:string"/> <xs:element name="preu" type="xs:decimal"/> <xs:element name="stock" type="xs:integer"/> </xs:sequence> <xs:attribute name="id" type="xs:ID" use="required"/> </xs:complexType> </xs:element> </xs:sequence> </xs:complexType> </xs:element> </xs:schema>

Exercici 4 - Cursos d'un centre

Enunciat
Modela un XML i DTD per a representar cursos d'un centre educatiu.
Condicions:
Un centre té diversos cursos.
Cada curs té:
nom
professor
hores
Pot tenir 0 o més alumnes.
<!ELEMENT centre (curs+)> <!ELEMENT curs (nom, professor, hores, alumne*)> <!ELEMENT nom (#PCDATA)> <!ELEMENT professor (#PCDATA)> <!ELEMENT hores (#PCDATA)> <!ELEMENT alumne (#PCDATA)>
<?xml version="1.0" encoding="UTF-8"?> <xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema"> <xs:element name="centre"> <xs:complexType> <xs:sequence> <xs:element name="curs" maxOccurs="unbounded"> <xs:complexType> <xs:sequence> <xs:element name="nom" type="xs:string"/> <xs:element name="professor" type="xs:string"/> <xs:element name="hores" type="xs:integer"/> <xs:element name="alumne" type="xs:string" minOccurs="0" maxOccurs="unbounded"/> </xs:sequence> </xs:complexType> </xs:element> </xs:sequence> </xs:complexType> </xs:element> </xs:schema>

Exercici 5 - Sistema de comandes

Defineix un XML i DTD per a representar comandes d'una botiga.
Condicions:
L'arrel és comandes.
Cada comanda té:
atribut numero
client
una o més linia
Cada línia conté:
producte
. quantitat
<!ELEMENT comandes (comanda+)> <!ELEMENT comanda (client, linia+)> <!ATTLIST comanda numero CDATA #REQUIRED> <!ELEMENT client (#PCDATA)> <!ELEMENT linia (producte, quantitat)> <!ELEMENT producte (#PCDATA)> <!ELEMENT quantitat (#PCDATA)>
<?xml version="1.0" encoding="UTF-8"?> <xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema"> <xs:element name="comandes"> <xs:complexType> <xs:sequence> <xs:element name="comanda" maxOccurs="unbounded"> <xs:complexType> <xs:sequence> <xs:element name="client" type="xs:string"/> <xs:element name="linia" maxOccurs="unbounded"> <xs:complexType> <xs:sequence> <xs:element name="producte" type="xs:string"/> <xs:element name="quantitat" type="xs:integer"/> </xs:sequence> </xs:complexType> </xs:element> </xs:sequence> <xs:attribute name="numero" type="xs:string" use="required"/> </xs:complexType> </xs:element> </xs:sequence> </xs:complexType> </xs:element> </xs:schema>

Exercici 6 - Biblioteca amb referències (ID / IDREF)

Modela un sistema de biblioteca.
Condicions:
Hi ha llibres i prestecs.
Cada llibre té un id.
Un préstec referencia el llibre mitjançant IDREF.
<!ELEMENT biblioteca (llibre+, prestec*)> <!ELEMENT llibre (titol)> <!ATTLIST llibre id ID #REQUIRED> <!ELEMENT titol (#PCDATA)> <!ELEMENT prestec (usuari)> <!ATTLIST prestec llibre IDREF #REQUIRED> <!ELEMENT usuari (#PCDATA)>
<?xml version="1.0" encoding="UTF-8"?> <xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema"> <xs:element name="biblioteca"> <xs:complexType> <xs:sequence> <xs:element name="llibre" maxOccurs="unbounded"> <xs:complexType> <xs:sequence> <xs:element name="titol" type="xs:string"/> </xs:sequence> <xs:attribute name="id" type="xs:ID" use="required"/> </xs:complexType> </xs:element> <xs:element name="prestec" minOccurs="0" maxOccurs="unbounded"> <xs:complexType> <xs:sequence> <xs:element name="usuari" type="xs:string"/> </xs:sequence> <xs:attribute name="llibre" type="xs:IDREF" use="required"/> </xs:complexType> </xs:element> </xs:sequence> </xs:complexType> </xs:element> </xs:schema>
